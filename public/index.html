GET /account-list

This end point send bank a list of expense and income categories that can be used to post transactions in endpoints further down the list.

Since this is general information, there are no mandatory fields and it does not require a request token.

POST /user/register

This endpoint is for registering a user for the first time.

It has 4 mandatory fields: username, email, password, confirmpassword

If any of the mandatory fields are missing, it will return an error.

If the passwords do not match, it will return an error.

An email with a verification code will be sent to the address provided. You must valdiate the email to proceed with login.

Required Request Headers: Content-Type: application/json

Example of data to send for request:

{ "username": "hello", "email": "hello@gamil.com", "password":"test", "confirmpassword":"test" }

Response Body Example:

"Registration Successfull"

POST /user/verify-email

This endpoint is to validate a user email when they register for the first time.

It has 2 mandatory fields: email, varificationCode

If any of the mandatory fields are missing, it will return an error.

You will not be able to proceed with the login until your email is verified.

Required Request Headers: Content-Type: application/json

Example of data to send for request:

{ "email": "hello@gamil.com", "verificationCode":"868698539" }

"Thank you! Your account is now verified."

POST /banks/add-bank-account

This endpoint is to set up the different types of payment methods

It has four mandatory endpoints: accType, accDesc, amount, balance_timestamp

You also require an authorization token to be able to use this end point. Reger to '/user/login' endpoint to get a toekn.

accType must be one of the follow: 'chequeing', 'credit-card', 'line-of-credit', 'savings', 'cash'

Account description field cannot be longer then 24 characters. And must be unique. If an account with the given description already exists, then the API will return an error.

you must set up at least one account to proceed with the remaining endpoints

The amount should be the balance at the time the relevant account is set up. This is to fascilitate cashflow features/endpoints.

Specify the date of the balance entered. (i.e. if you are using your savings statment balance, then use the statement end date.)

Please do not include any sensitive account information in the description header. This is only to identify between multiple accounts if you set up more then one.

If you are missing any of the fields or there is a problem with the fields, the api will return a 400 response with some information regarding what went wrong.

Required Request Headers: Content-Type: application/json

Authorization: Bearer 'JWT Token'

Example of data to send for request:

{ "accType": "credit-card", "accDesc":"Walmart Rewards", "amount": "0", "balance_timestamp": "Apr 1, 2022" }

Response Body Example:

"Account added Successfully"

DELETE /banks/single-account

This endpoint searches for banks by id. It then searches the transaction and budget records to see if it has been used in the past. If it hasn't been used then the bank account is deleted.

There is 1 mandatory field: bankid

An error is returned if the field is missing.

If the the user does not have a bank account with the specified id, then no records will be deleted.

If the bank account has been used in the past in transaction or budget records, then no records will be deleted.

Required Request Headers: Content-Type: application/json

Authorization: Bearer 'JWT Token'

Example of parameters to send for request:

{ "bankid": "5", }

Alternatively add params to end of URL

'/banks/single-account?bankid=5

Response Body Example:

"1 record deleted"

POST /transfer/single

This endpoint allows you to add transfer records for a specific user. This is the movement between payment account. (i.e. a credit card payment using your chequeing account would fall under this category)

It has 4 mandatory fields: amount, debit, credit, transaction_timestamp

There is an optional field 'description' in which you can add details about the transaction if you want.

Only Payment accounts can be used for debit or credit.

Note: This API does not support records older then Jan 1, 2022.

If any of the mandatory fields are missing, then the endpoint will retunr an error with some information about what went wrong.

Required Request Headers: Content-Type: application/json

Authorization: Bearer 'JWT Token'

Example of data to send for request:

{ "debit": "RBC", "credit":"TD Bank", "transaction_timestamp": "May 1, 2022", "amount": "200" }

[ { "id": 1, "amount": 200, "Debit": "RBC", "Credit": "TD Bank", "Description": "", "Record_timestamp": 1657680197080, "Transaction_timestamp": 1651377600000, "user_id": 1 } ]

PATCH /transfer/single

This endpoint searches for the transfer by id and updates the parameters provided.

There is 1 mandatory field: tranid

The following optional fields can be updated: amount, debit, credit, description, transaction_timestamp. At least one of these optional fields must be provided with the tranid.

Note that Debit and Credit cannot be the same account. (i.e. If you provide debit only for update, and it is the same as the credit for the transfer, then the update will be rejected.) If you provide both debit and credit for update, then previous values of the transfer will not be checked.

The API does not support maintianing records prior to Jan 1, 2022.

The API will respond back with the updated information for the transaction in question for a successfull request.

Required Request Headers: Content-Type: application/json

Authorization: Bearer 'JWT Token'

Example of body to send for request:

{ "tranid": "1", "amount": "50", }

Response Body Example:

[ { "id": 1, "amount": 50, "Debit": "RBC", "Credit": "TD Bank", "Description": "", "Record_timestamp": 1657680046193, "Transaction_timestamp": 1651377600000, "user_id": 1 } ]
